# 毕业设计项目概览：体育馆预约与社交拼场系统 2.0

> **项目定位**：基于“协同消费”理念的智慧体育馆管理系统，旨在平衡体育场馆利用率与用户社交预约需求。

---

## 1. 核心研究背景与意义 (Thesis Background)

在传统的场馆预约系统中，用户往往面临“凑不够人”或“场地浪费”的尴尬。本项目通过引入**拼场（Sharing）机制**，利用数字化手段实现场地资源的精细化流转，不仅提高了场馆端的运营效率，也降低了用户的运动门槛。

---

## 2. 系统技术架构 (System Architecture)

系统采用典型的**前后端分离**架构，并针对小程序移动端环境进行了特定优化：

### 2.1 技术栈总览
- **前端 (Frontend)**: 
  - 核心框架：`uni-app` + `Vue 3` (Composition API)
  - 状态管理：`Pinia` (轻量化、类型安全)
  - 通信机制：`WebSocket` (支持订单状态实时推送)
- **后端 (Backend)**:
  - 核心框架：`Spring Boot 3`
  - 持久化层：`Spring Data JPA` + `MySQL`
  - 权限安全：`Spring Security` + `Stitch JWT` + 微信 OAuth 2.0
- **运维与架构**:
  - RESTful API 规范化设计
  - 请求队列算法 (解决单线程并发限制)

---

## 3. 核心业务逻辑设计 (Core Implementation)

### 3.1 拼场逻辑状态机 (State Machine)
这是本项目论文的最大亮点。系统设计了严密的订单迁移状态，确保资金与场地的绝对一致。
- **状态流转**：`PENDING` -> `OPEN` (发起支付) -> `APPROVED_PENDING_PAYMENT` (获准加入) -> `SHARING_SUCCESS` (成团) -> `VERIFIED` (核销)。
- **冲突处理**：当场地被“独享预约”锁定后，关联的所有拼场请求将自动触发逻辑退款并释放。

### 3.2 智能生成与排期引擎 (TimeSlot Engine)
- **滚动窗口机制**：系统自动维持未来 7 天的可用时间段（TimeSlot）。
- **粒度控制**：支持按场馆类型设置不同的时间片（通常为 60/90/120 分钟）。
