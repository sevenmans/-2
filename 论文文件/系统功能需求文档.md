# 系统功能需求文档

## 1. 系统用户角色

本系统主要包含以下两类用户角色：

### 1.1 普通用户 (User)

- **角色标识**: `ROLE_USER`
- **描述**: 系统的终端消费者，通过微信小程序访问系统。
- **核心职责**: 查找场馆、预订场地、参与社交拼场、支付费用、出示凭证入场。

### 1.2 管理员 (Administrator)

- **角色标识**: `ROLE_VENUE_ADMIN` 
- **描述**: 场馆的运营与管理者，通过微信小程序（管理员视图）访问系统。
- **核心职责**: 维护场馆信息、监控预订情况、处理订单异常、现场核销入场、查看经营数据。

***

## 2. 核心功能模块

为提升系统功能的内聚性与管理效率，我们将原有的细分功能点融合为 **10个核心功能模块**：

### 2.1 用户认证与管理

**功能描述**：提供安全便捷的身份认证服务，支持微信一键授权，确保用户身份唯一性，并提供个人资料（昵称、头像）的维护能力。
**包含功能**：

- 微信一键登录 
- 个人信息修改（头像、昵称、手机号）

```
graph TD
    User((普通用户))
    UC1(微信一键登录)
    UC2(手机号登录)
    UC3(修改个人资料)
    
    User --> UC1
    User --> UC2
    User --> UC3
```

### 2.2 场馆浏览与查询

**功能描述**：为用户提供多维度的场馆检索服务，支持按运动类型、价格、区域筛选场馆，展示高清设施图片、营业信息及实时场地排期状态。
**包含功能**：

- 场馆列表筛选与排序
- 场馆详情查看（设施、地址）
- 实时场地排期查询（可用/占用/维护）

```mermaid
usecaseDiagram
    actor User as "普通用户"
    usecase UC1 as "筛选场馆"
    usecase UC2 as "查看详情与导航"
    usecase UC3 as "查询实时排期"
    
    User --> UC1
    User --> UC2
    User --> UC3
```

### 2.3 场地包场预约

**功能描述**：支持用户选择空闲时间段进行全场预订，完成订单提交与支付流程，并在规定时间内未支付自动释放场地，支持符合规则的订单取消与退款。
**包含功能**：

- 选择时间段提交订单
- 在线支付（微信支付模拟）
- 订单自动过期处理
- 取消订单与退款

```mermaid
usecaseDiagram
    actor User as "普通用户"
    actor Payment as "支付系统"
    usecase UC1 as "提交包场订单"
    usecase UC2 as "在线支付"
    usecase UC3 as "取消订单退款"
    
    User --> UC1
    UC1 --> UC2
    UC2 --> Payment
    User --> UC3
```

### 2.4 拼场社交互动

**功能描述**：提供基于场地的社交约球服务，支持用户发起AA制拼场活动、招募球友，以及浏览拼场大厅申请加入他人队伍，涵盖申请审核与份额支付全流程。
**包含功能**：

- 浏览拼场大厅
- 发起拼场（设置费用/人数）
- 申请加入与审核
- 支付个人拼场份额

```mermaid
usecaseDiagram

    actor Joiner as "参与者"
    usecase UC1 as "发起拼场招募"
    usecase UC2 as "申请加入"

    Creator --> UC1

    Joiner --> UC2

    UC3 ..> UC4 : 通过后

```

### 2.5 订单与凭证管理

**功能描述**：集中管理用户的历史订单记录，提供按状态筛选查询功能，并为已支付订单生成唯一的入场核销凭证（数字码）。
**包含功能**：

- 我的订单列表（全部/待支付/已完成等）
- 查看订单详情
- 展示入场核销码

```mermaid
usecaseDiagram
    actor User as "普通用户"
    usecase UC1 as "查看订单列表"
    usecase UC2 as "查看详情"
    usecase UC3 as "展示核销码"
    
    User --> UC1
    UC1 --> UC2
    UC2 --> UC3
```

### 2.6 管理员工作台

**功能描述**：为管理员提供经营数据的可视化概览，实时展示今日关键指标（订单量、营收、待核销数），辅助运营决策。
**包含功能**：

- 查看今日订单统计
- 查看今日营收数据
- 查看待核销任务数

```mermaid
usecaseDiagram
    actor Admin as "管理员"
    usecase UC1 as "查看经营数据"
    usecase UC2 as "监控待办事项"
    
    Admin --> UC1
    Admin --> UC2
```

### 2.7 现场核销验证

**功能描述**：支持管理员在场馆现场通过扫描或输入用户核销码，快速验证订单有效性并完成入场登记，防止伪造凭证。
**包含功能**：

- 扫描/输入核销码
- 验证订单有效性
- 确认核销入场

```mermaid
usecaseDiagram

    usecase UC2 as "验证并核销"
    
    Admin --> UC1
    UC1 --> UC2
```

### 2.8 场馆资源管理

**功能描述**：支持管理员对场馆的基础信息进行全生命周期管理，包括场馆的新增、编辑、上下架及价格策略调整。
**包含功能**：

- 新增/编辑场馆信息
- 场馆上下架控制
- 价格与营业时间设置

```mermaid

```

### 2.9 排期与库存控制

**功能描述**：提供灵活的场地库存管理能力，管理员可手动锁定特定时间段（设为占用或维护中），处理设备维修或临时征用等突发情况。
**包含功能**：

- 查看全馆排期表
- 锁定时间段（设为占用）
- 设置维护状态（设为维护中）
- 释放时间段

### 2.10 全系统订单监管

**功能描述**：赋予管理员对平台所有订单的监管权限，支持多维度检索历史订单，并对违规或异常占位订单进行强制取消与退款处理。
**包含功能**：

- 全量订单检索与查询
- 查看订单详细日志
- 强制取消异常订单
