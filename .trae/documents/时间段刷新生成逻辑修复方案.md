# 体育馆预约系统 - 时间段刷新生成逻辑修复方案

## 1. 项目现状分析

### 1.1 当前架构概览

**技术栈**:
- 前端: Vue3 + uni-app (微信小程序)
- 后端: Spring Boot + MySQL + JPA
- 状态管理: Vuex
- 缓存: 多层缓存策略

**核心文件结构**:
```
前端:
├── stores/venue.js                    # 主要状态管理
├── api/timeslot.js                   # API接口层
├── utils/timeslot-manager.js         # 时间段管理器
├── utils/timeslot-refresh-optimizer.js # 刷新优化器
└── components/TimeSlot.vue           # 时间段组件

后端:
├── TimeSlotController.java           # 控制器层

└── TimeSlot.java                    # 实体类
```

### 1.2 存在的主要问题

#### 1.2.1 架构层面问题

**问题1: 功能重叠和代码冗余**
- `venue.js` 中包含大量时间段处理逻辑
- `timeslot-manager.js` 和 `timeslot-refresh-optimizer.js` 功能重叠
- 多个文件都有缓存管理逻辑
- 状态处理逻辑分散在多个地方

**问题2: 缓存策略混乱**
- 前端有多层缓存：Vuex、localStorage、request.js缓存
- 缓存清理策略不统一
- 缓存键命名不规范
- 缓存过期时间设置不合理

**问题3: 状态同步复杂**
- 前后端状态字段不统一
- 状态更新触发点过多
- 乐观更新和悲观更新混用
- WebSocket和轮询并存

#### 1.2.2 性能层面问题

**问题4: 刷新频率控制不当**
- 缺乏有效的防抖和节流机制
- 多个组件可能同时触发刷新
- 刷新粒度过细，影响性能

**问题5: 错误处理分散**
- 错误处理逻辑分散在各个文件
- 降级策略不够完善
- 缺乏统一的错误监控

#### 1.2.3 维护层面问题

**问题6: 代码可读性差**
- 方法命名不够清晰
- 业务逻辑和技术逻辑混合
- 缺乏统一的代码规范

## 2. 修复方案选择

### 2.1 方案对比

#### 方案A: 渐进式修复（推荐）
**优点**:
- 风险较低，可以逐步验证效果

#### 方案B: 完全重构
**优点**:

- 代码结构更加清晰
- 性能优化更彻底

**缺点**:

```javascript
// utils/unified-timeslot-manager.js
class UnifiedTimeSlotManager {
  constructor() {
    this.cache = new Map()
    this.refreshQueue = new Map()
    this.lastRefreshTimes = new Map()
    this.refreshIntervals = {
      current: 30000,   // 当前日期30秒
      adjacent: 60000,  // 相邻日期1分钟
      future: 300000    // 未来日期5分钟
    }
  }

  // 统一的获取时间段方法
  async getTimeSlots(venueId, date, options = {}) {
    const { forceRefresh = false, loading = true } = options
    const cacheKey = `${venueId}_${date}`
    
    // 检查缓存
    if (!forceRefresh && this.cache.has(cacheKey)) {
      const cached = this.cache.get(cacheKey)
      if (this.isCacheValid(cached)) {
        return cached.data
      }
    }
    
    // 防重复请求
    if (this.refreshQueue.has(cacheKey)) {
      return this.refreshQueue.get(cacheKey)
    }
    
    // 发起请求
    const promise = this.fetchTimeSlots(venueId, date, { loading })
    this.refreshQueue.set(cacheKey, promise)
    
    try {
      const result = await promise
      this.updateCache(cacheKey, result)
      return result
    } finally {
      this.refreshQueue.delete(cacheKey)
    }
  }

  // 统一的状态处理方法
  processTimeSlotStatus(timeSlots, date) {
    const now = new Date()
    const currentDate = now.toISOString().split('T')[0]
    
    return timeSlots.map(slot => {
      // 标准化状态字段
      this.normalizeSlotStatus(slot)
      
      // 过期检查
      if (date === currentDate) {
        this.checkSlotExpiry(slot, now)
      }
      
      // 设置UI属性
      this.setSlotUIProperties(slot)
      
      return slot
    })
  }

  // 智能刷新调度
  scheduleRefresh(venueId, date) {
    const key = `${venueId}_${date}`
    const interval = this.getRefreshInterval(date)
    const lastRefresh = this.lastRefreshTimes.get(key) || 0
    const now = Date.now()
    
    if (now - lastRefresh >= interval) {
      this.executeRefresh(venueId, date)
      this.lastRefreshTimes.set(key, now)
    }
  }
}

export default new UnifiedTimeSlotManager()
```

### 3.2 第二阶段：优化缓存策略

#### 3.2.1 建立统一的缓存管理

**创建 `utils/cache-manager.js`**:

```javascript
// utils/cache-manager.js
class CacheManager {
  constructor() {
    this.memoryCache = new Map()
    this.cacheExpiry = new Map()
    this.storagePrefix = 'timeslot_'
  }
  
  // 设置缓存
  set(key, data, ttl = 300000) {
    const expiry = Date.now() + ttl
    
    // 内存缓存
    this.memoryCache.set(key, data)
    this.cacheExpiry.set(key, expiry)
    
    // 本地存储缓存（持久化）
    try {
      uni.setStorageSync(this.storagePrefix + key, {
        data,
        expiry
      })
    } catch (error) {
      console.warn('本地存储失败:', error)
    }
  }
  
  // 获取缓存
  get(key) {
    // 先检查内存缓存
    const expiry = this.cacheExpiry.get(key)
    if (expiry && Date.now() < expiry) {
      return this.memoryCache.get(key)
    }
    
    // 检查本地存储
    try {
      const stored = uni.getStorageSync(this.storagePrefix + key)
      if (stored && stored.expiry > Date.now()) {
        // 恢复到内存缓存
        this.memoryCache.set(key, stored.data)
        this.cacheExpiry.set(key, stored.expiry)
        return stored.data
      }
    } catch (error) {
      console.warn('读取本地存储失败:', error)
    }
    
    return null
  }
  
  // 清除缓存
  clear(pattern) {
    if (pattern) {
      // 模式匹配清除
      for (const key of this.memoryCache.keys()) {
        if (key.includes(pattern)) {
          this.delete(key)
        }
      }
    } else {
      // 清除所有
      this.memoryCache.clear()
      this.cacheExpiry.clear()
    }
  }
  
  delete(key) {
    this.memoryCache.delete(key)
    this.cacheExpiry.delete(key)
    try {
      uni.removeStorageSync(this.storagePrefix + key)
    } catch (error) {
      console.warn('删除本地存储失败:', error)
    }
  }
}

export default new CacheManager()
```

#### 3.2.2 统一缓存策略

**缓存层次结构**:
1. **L1缓存**: 内存缓存（最快，页面刷新丢失）
2. **L2缓存**: 本地存储（持久化，容量有限）
3. **L3缓存**: 服务端缓存（通过API控制）

**缓存TTL策略**:
- 当天数据：1分钟
- 未来3天数据：5分钟
- 其他未来数据：15分钟
- 历史数据：1小时

```javascript
// 增量更新逻辑
class IncrementalUpdater {
  updateTimeSlots(oldSlots, newSlots) {
    const changes = []
    const oldSlotsMap = new Map(oldSlots.map(slot => [slot.id, slot]))
    
    newSlots.forEach(newSlot => {
      const oldSlot = oldSlotsMap.get(newSlot.id)
      
      if (!oldSlot) {
        // 新增的时间段
        changes.push({ type: 'add', slot: newSlot })
      } else if (this.hasStatusChanged(oldSlot, newSlot)) {
        // 状态发生变化的时间段
        changes.push({ type: 'update', slot: newSlot, oldSlot })
      }
    })
    
    return changes
  }
  
  hasStatusChanged(oldSlot, newSlot) {
    return oldSlot.status !== newSlot.status ||
           oldSlot.price !== newSlot.price ||
           oldSlot.orderId !== newSlot.orderId
  }
}
```

```javascript
// utils/error-handler.js
class ErrorHandler {
  constructor() {
    this.errorCounts = new Map()
    this.maxRetries = 3
    this.retryDelay = 1000
  }
  
  async handleTimeSlotError(error, context) {
    const errorKey = `${context.venueId}_${context.date}`
    const count = this.errorCounts.get(errorKey) || 0
    
    console.error('时间段操作失败:', error, context)
    
    // 错误分类处理
    if (this.isNetworkError(error)) {
      return this.handleNetworkError(error, context, count)
    } else if (this.isServerError(error)) {
      return this.handleServerError(error, context, count)
    } else {
      return this.handleUnknownError(error, context)
    }
  }
  
  async handleNetworkError(error, context, retryCount) {
    if (retryCount < this.maxRetries) {
      // 指数退避重试
      const delay = this.retryDelay * Math.pow(2, retryCount)
      await this.sleep(delay)
      
      this.errorCounts.set(`${context.venueId}_${context.date}`, retryCount + 1)
      return { shouldRetry: true, delay }
    } else {
      // 降级到缓存数据
      return this.fallbackToCachedData(context)
    }
  }
  
  async fallbackToCachedData(context) {
    // 尝试从缓存获取数据
    const cachedData = cacheManager.get(`${context.venueId}_${context.date}`)
    if (cachedData) {
      uni.showToast({
        title: '网络异常，显示缓存数据',
        icon: 'none'
      })
      return { data: cachedData, fromCache: true }
    }
    
    // 生成默认数据
    return this.generateDefaultTimeSlots(context)
  }
}

export default new ErrorHandler()
```

### 3.5 第五阶段：性能优化

#### 3.5.1 实现防抖和节流

**创建 `utils/performance-optimizer.js`**:

```javascript
// utils/performance-optimizer.js
class PerformanceOptimizer {
  constructor() {
    this.debounceTimers = new Map()
    this.throttleTimers = new Map()
  }
  
  // 防抖：延迟执行，重复调用会重置计时器
  debounce(key, func, delay = 300) {
    if (this.debounceTimers.has(key)) {
      clearTimeout(this.debounceTimers.get(key))
    }
    
    const timer = setTimeout(() => {
      func()
      this.debounceTimers.delete(key)
    }, delay)
    
    this.debounceTimers.set(key, timer)
  }
  
  // 节流：限制执行频率
  throttle(key, func, interval = 1000) {
    if (this.throttleTimers.has(key)) {
      return false // 正在节流中
    }
    
    func()
    
    const timer = setTimeout(() => {
      this.throttleTimers.delete(key)
    }, interval)
    
    this.throttleTimers.set(key, timer)
    return true
  }
  
  // 批量操作优化
  batchOperation(operations, batchSize = 10) {
    return new Promise((resolve) => {
      let index = 0
      const results = []
      
      const processBatch = () => {
        const batch = operations.slice(index, index + batchSize)
        
        Promise.all(batch.map(op => op())).then(batchResults => {
          results.push(...batchResults)
          index += batchSize
          
          if (index < operations.length) {
            // 使用 requestAnimationFrame 避免阻塞UI
            requestAnimationFrame(processBatch)
          } else {
            resolve(results)
          }
        })
      }
      
      processBatch()
    })
  }
}

export default new PerformanceOptimizer()
```

```javascript
// 简化的API方法
export function getVenueTimeSlots(venueId, date, options = {}) {
  const { forceRefresh = false, loading = true } = options
  
  const requestOptions = {
    loading,
    cache: !forceRefresh,
    cacheTTL: forceRefresh ? 0 : getCacheTTL(date)
  }
  
  if (forceRefresh) {
    requestOptions.params = {
      _t: Date.now(),
      _nocache: 1
    }
    requestOptions.headers = {
      'Cache-Control': 'no-cache, no-store, must-revalidate'
    }
  }
  
  return get(`/timeslots/venue/${venueId}/date/${date}`, 
    requestOptions.params || {}, 
    requestOptions
  )
}

// 根据日期计算缓存TTL
function getCacheTTL(date) {
  const today = new Date().toISOString().split('T')[0]
  if (date === today) return 60000      // 当天1分钟
  if (date < today) return 3600000      // 历史1小时
```

```javascript
import { SLOT_STATUS } from './timeslot-constants.js'

export class TimeSlotValidator {
  // 验证时间段数据格式
  static validateTimeSlot(slot) {
    const errors = []
    
    if (!slot.id) errors.push('缺少时间段ID')
    if (!slot.startTime) errors.push('缺少开始时间')
    if (!slot.endTime) errors.push('缺少结束时间')
    if (!slot.venueId) errors.push('缺少场馆ID')
    if (!slot.date) errors.push('缺少日期')
    
    // 验证状态值
    if (!Object.values(SLOT_STATUS).includes(slot.status)) {
      errors.push(`无效的状态值: ${slot.status}`)
    }
    
    // 验证时间格式
    if (!this.isValidTimeFormat(slot.startTime)) {
      errors.push(`无效的开始时间格式: ${slot.startTime}`)
    }
    
    if (!this.isValidTimeFormat(slot.endTime)) {
      errors.push(`无效的结束时间格式: ${slot.endTime}`)
    }
    
    return errors
  }
  
  // 验证时间格式 (HH:mm)
  static isValidTimeFormat(time) {
    return /^([01]?[0-9]|2[0-3]):[0-5][0-9]$/.test(time)
  }
  
  // 标准化时间段数据
  static normalizeTimeSlot(slot) {
    const normalized = { ...slot }
    
    // 处理不同的状态字段名
    if (slot.available !== undefined) {
      normalized.status = slot.available ? SLOT_STATUS.AVAILABLE : SLOT_STATUS.OCCUPIED
    }
    
    // 确保必要字段存在
    normalized.disabled = normalized.disabled || false
    normalized.price = normalized.price || 0
    
    // 验证并修复无效状态
    if (!Object.values(SLOT_STATUS).includes(normalized.status)) {
      normalized.status = SLOT_STATUS.OCCUPIED
    }
    
    return normalized
  }
}
```
