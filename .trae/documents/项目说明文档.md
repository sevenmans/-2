# 体育馆预约系统 - 项目说明文档

## 📋 项目概述

**项目名称**: 体育馆预约系统微信小程序
**开发周期**: 2025年6月 - 至今（持续优化中）
**项目类型**: 全栈开发项目
**团队规模**: 个人独立开发

这是一个基于微信小程序的体育馆预约系统，为用户提供便捷的场馆预约服务。系统支持独享预约和拼场功能，具备完整的用户管理、订单管理、支付集成等核心业务模块。项目采用前后端分离架构，具有高可用性、高性能的特点。

**商业价值**:

* 解决传统体育馆预约效率低下的痛点

* 提升场馆利用率，降低运营成本

* 为用户提供便捷的移动端预约体验

* 支持拼场功能，增强用户社交属性

## 🏗️ 技术架构

### 整体架构设计

```
微信小程序端 (uni-app + Vue3)
        ↓
    API网关层
        ↓
Spring Boot后端服务
        ↓
    MySQL数据库
```

### 前端技术栈

* **框架**: uni-app + Vue 3 + Composition API

* **状态管理**: Pinia (从Vuex迁移，提升性能)

* **UI组件**: 自定义组件库 + uni-ui

* **开发工具**: HBuilderX + 微信开发者工具

* **构建工具**: Vite

* **代码规范**: ESLint + Prettier

### 后端技术栈

* **框架**: Spring Boot 3.4.5

* **数据库**: MySQL 8.0 + JPA/Hibernate

* **安全认证**: Spring Security + JWT

* **API文档**: SpringDoc OpenAPI 3.0

* **构建工具**: Maven

* **Java版本**: JDK 17

### 核心技术特性

* **响应式设计**: 适配多种屏幕尺寸

* **实时数据同步**: WebSocket + 轮询机制

* **缓存策略**: 多层缓存优化性能

* **并发控制**: 请求队列管理，防止超时

* **错误处理**: 完善的异常处理机制

## 🚀 核心功能模块

### 1. 用户认证与权限管理

* **微信授权登录**: 集成微信小程序登录API

* **JWT令牌管理**: 无状态身份验证

* **用户信息管理**: 个人资料编辑、头像上传

* **权限控制**: 基于角色的访问控制(RBAC)

### 2. 场馆管理系统

* **场馆信息展示**: 详细的场馆信息、设施介绍

* **实时状态更新**: 可用、维护中、已占用三种状态

* **场馆搜索筛选**: 多维度筛选功能

* **图片展示**: 场馆图片轮播展示

### 3. 智能预约系统

* **时间段管理**: 动态生成和管理时间段

* **冲突检测**: 防止重复预约和时间冲突

* **状态实时同步**: 预约状态实时更新

* **预约历史**: 完整的预约记录管理

* **取消机制**: 灵活的取消和退款策略

### 4. 拼场社交功能

* **创建拼场**: 发起拼场邀请，设置参与条件

* **申请管理**: 申请审核、自动匹配

* **实时通讯**: 拼场成员间的消息通知

* **费用分摊**: 自动计算分摊费用

### 5. 订单与支付系统

* **订单生成**: 自动生成订单，支持多种预约类型

* **支付集成**: 微信支付API集成

* **订单状态管理**: 完整的订单生命周期管理

* **退款处理**: 自动化退款流程
