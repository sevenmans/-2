# 体育馆预约系统 - 数据字段对照文档

## 文档说明
本文档定义了体育馆预约系统中所有实体类的字段规范，前端开发必须严格按照此文档中的字段名称、类型和含义进行数据交互。

---

## 1. 用户实体 (User)

| 字段名 | 类型 | 含义 | 是否必填 | 备注 |
|--------|------|------|----------|------|
| id | Long | 用户ID | 是 | 主键，自动生成 |
| username | String | 用户名 | 是 | 唯一，长度3-50字符 |
| password | String | 密码 | 是 | 加密存储 |
| nickname | String | 昵称 | 否 | 显示名称 |
| phone | String | 手机号 | 否 | 唯一，用于登录 |
| email | String | 邮箱 | 否 | 唯一，用于登录 |
| avatar | String | 头像URL | 否 | 用户头像链接 |
| roles | Set<Role> | 用户角色 | 是 | 角色集合 |
| active | Boolean | 账户状态 | 是 | true=激活，false=禁用 |
| createdAt | LocalDateTime | 创建时间 | 是 | 自动生成 |
| updatedAt | LocalDateTime | 更新时间 | 是 | 自动更新 |

### 角色枚举 (Role)
- `ROLE_USER`: 普通用户
- `ROLE_ADMIN`: 管理员
- `ROLE_VENUE_ADMIN`: 场馆管理员
- `ROLE_SUPER_ADMIN`: 超级管理员

## 前端开发规范

### 1. 字段命名规范
- 前端必须使用与后端完全一致的字段名
- 所有时间字段统一使用ISO 8601格式
- 枚举值必须使用后端定义的确切值

### 2. 数据类型对应
- `Long` → `number`
- `String` → `string`
- `Boolean` → `boolean`
- `BigDecimal` → `number` (注意精度处理)
- `LocalDateTime` → `string` (ISO 8601格式)
- `LocalDate` → `string` (YYYY-MM-DD格式)
- `LocalTime` → `string` (HH:mm:ss格式)

### 3. 必填字段验证
- 前端表单必须对所有必填字段进行验证
- 提交数据前必须确保必填字段不为空

### 4. 枚举值处理
- 所有枚举值必须与后端保持一致
- 前端显示时可以进行本地化处理，但提交给后端的值必须是英文枚举值

### 5. 时间处理规范
- 所有时间字段在前端显示时需要进行格式化
- 提交给后端时必须转换为对应的格式
- 注意时区处理
