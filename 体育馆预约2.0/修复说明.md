## 修复的问题

### 问题1: 界面中出现大量 `&gt;` 字符串

**问题原因:**
在Vue模板中直接使用 `>` 符号作为箭头时,在某些情况下会被HTML实体编码为 `&gt;`,导致界面显示异常。

**解决方案:**
将所有模板中的 `>` 符号替换为Unicode字符 `›` (单右尖括号,U+203A),这是一个专门用于导航箭头的字符,不会被HTML编码。

**修复的文件:**
1. `src/pages/index/index.vue` - 首页的"更多"链接
2. `src/pages/user/profile.vue` - 我的页面的菜单箭头
3. `src/pages/booking/detail.vue` - 预约详情页的联系信息箭头
4. `src/pages/sharing/detail.vue` - 拼场详情页的场馆卡片箭头
5. `src/pages/venue/detail.vue` - 场馆详情页CSS中的箭头

**修复详情:**
- 首页: 2处 "更多 >" → "更多 ›"
- 我的页面: 6处菜单项箭头 ">" → "›"
- 预约详情: 2处联系信息箭头 ">" → "›"
- 拼场详情: 1处场馆卡片箭头 ">" → "›"
- 场馆详情: 1处CSS伪元素箭头 content: '>' → content: '›'

## 学习要点

### 1. HTML实体编码问题
**原因:** 
- 在HTML/XML中,`<` 和 `>` 是特殊字符,用于标记标签
- Vue模板编译时可能会将这些字符转义为HTML实体 (`&lt;` 和 `&gt;`)

**解决方法:**
- 使用Unicode字符代替: `›` (U+203A) 或 `‹` (U+2039)
- 或者使用HTML实体: `&rsaquo;` 或 `&lsaquo;`
- 或者使用emoji: `▶` 或 `◀`

### 2. 开发环境与生产环境分离
**最佳实践:**
- 使用环境变量控制调试功能的显示
- 使用条件编译 (`#ifdef` / `#ifndef`) 在编译时移除调试代码
- 将调试工具放在独立的页面或模块中,不要混入正常业务页面

**示例:**
```vue
<!-- 仅在开发环境显示 -->
<view v-if="process.env.NODE_ENV === 'development'" class="dev-toolbar">
  <!-- 调试按钮 -->
</view>
```

### 3. 代码整洁原则
- **单一职责**: 每个页面应该只负责一个主要功能
- **最小化**: 只保留必要的功能和代码
- **可维护性**: 代码应该易于理解和修改
