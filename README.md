# 体育馆预约系统

## 📋 项目概述

这是一个基于微信小程序的体育馆预约系统，用户可以通过小程序预约体育场馆的时间段，支持独享预约和拼场功能。

## 🏗️ 技术架构

### 前端技术栈
- **框架**: uni-app + Vue 3
- **状态管理**: Pinia (已从 Vuex 迁移)
- **开发工具**: HBuilderX + 微信开发者工具
- **平台**: 微信小程序

### 后端技术栈
- **框架**: Spring Boot
- **数据库**: MySQL
- **认证**: JWT Token
- **API**: RESTful API

## 🚀 核心功能

### 1. 用户认证系统
- **登录/注册**: 支持微信授权登录
- **用户信息管理**: 个人资料编辑、头像上传
- **权限控制**: 基于 JWT 的身份验证

### 2. 场馆管理
- **场馆列表**: 展示所有可用场馆
- **场馆详情**: 查看场馆信息、设施、价格
- **场馆搜索**: 按名称、类型筛选场馆

### 3. 预约系统
- **时间段选择**: 查看可用时间段
- **预约状态**: 可用、维护中、已占用三种状态
- **预约管理**: 创建、查看、取消预约
- **预约历史**: 查看历史预约记录

### 4. 拼场功能
- **创建拼场**: 发起拼场邀请
- **参与拼场**: 申请加入他人拼场
- **拼场管理**: 审核申请、管理参与者

### 5. 订单系统
- **订单创建**: 生成预约订单
- **支付集成**: 支持微信支付
- **订单状态**: 待支付、已支付、已完成、已取消

## 📱 使用说明

### 开发环境搭建
1. **安装 HBuilderX**: 下载并安装最新版本
2. **安装微信开发者工具**: 用于小程序调试
3. **导入项目**: 在 HBuilderX 中打开项目文件夹
4. **配置后端**: 确保后端服务运行在 `http://localhost:8080`

### 运行项目
1. 在 HBuilderX 中选择"运行" -> "运行到小程序模拟器" -> "微信开发者工具"
2. 微信开发者工具会自动打开并加载项目
3. 在模拟器中测试各项功能

### 配置说明
- **API 基础地址**: `config/index.js` 中的 `baseURL`
- **请求超时**: 默认 10 秒，可在 `utils/request.js` 中调整
- **并发控制**: `utils/request-queue.js` 中的 `maxConcurrentRequests`

## 🛠️ 工具函数说明

### RequestQueue (请求队列管理)
**位置**: `utils/request-queue.js`
**用途**: 控制并发请求数量，避免同时发起过多请求导致超时

**使用方法**:
```javascript
import { requestQueue } from '@/utils/request-queue.js'

// 添加请求到队列
await requestQueue.add(
  () => api.getUserInfo(), // 请求函数
  (result) => { /* 处理结果 */ }, // 成功回调
  (error) => { /* 处理错误 */ } // 错误回调
)
```

**参数说明**:
- `requestFn`: 返回 Promise 的请求函数
- `handler`: 成功时的处理函数
- `errorHandler`: 失败时的处理函数 (可选)

### TimeSlotManager (时间段管理器)
**位置**: `utils/timeslot-manager.js`
**用途**: 统一管理时间段的获取、生成、同步逻辑

**主要方法**:
- `getTimeSlots(venueId, date)`: 获取指定场馆和日期的时间段
- `generateSyncAndRefetch(venueId, date)`: 生成时间段并同步到后端
- `analyzeSlotStatus(slots)`: 分析时间段状态分布

## 🐛 问题排查

### 常见问题
1. **请求超时**: 检查网络连接和后端服务状态
2. **登录失败**: 确认 JWT Token 是否有效
3. **页面空白**: 检查控制台错误信息
4. **数据不更新**: 清除缓存或强制刷新

### 调试工具
- **控制台日志**: 查看详细的请求和响应信息
- **网络面板**: 监控 API 调用情况
- **存储面板**: 检查本地存储的数据

## 📈 性能优化建议

1. **图片优化**: 使用适当尺寸的图片，启用懒加载

3. **请求优化**: 避免重复请求，使用请求队列控制并发
4. **代码分割**: 按需加载页面和组件
